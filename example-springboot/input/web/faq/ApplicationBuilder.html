What is the Application Builder?

Application code is just formatted texts with certain rules. The application builder generate code by some structured data and templates.

This was the starting point for elastic objects some years ago. I created java code, tests, hibernate configuration or sql by simple excel sheet with some templates.

This help avoiding the boring part of programming by generating the repeating structures.

Part of ElasticObjects like #{ConfigLinkCall->ModelConfig, AnObject}. are itself generated by templates and #{GithubLinkCall->NONE, B|calls.generate|GenerateJavaConfigCall.java}.. Here the generated sources serve as template itself keep control calls within with "keepCall". Outside the call directives one can edit code.

Actually there are three templates for generating java code:

  • #{ConfigLinkCall->ModelConfig, Call}. by #{GithubLinkCall->NONE, BR|templates.java.interface|INTERFACECreate.tpl}.
  • #{ConfigLinkCall->ModelConfig, CallImpl}. by #{GithubLinkCall->NONE, BR|templates.java.bean|BEANCreate.tpl}.
  • #{ConfigLinkCall->ModelConfig, ConfigImpl}. by #{GithubLinkCall->NONE, BR|templates.java.config|CONFIGCreate.tpl}.

A control template combine loading data and generation call by #{GithubLinkCall->NONE, BR|templates.java|JavaBuilder.tpl}.

Furthermore the json configurations are generated by #{GithubLinkCall->NONE, B|calls.generate|GenerateJsonCall.java}.. The control template is #{GithubLinkCall->NONE, BR|templates.json|JsonBuilder.tpl}.
