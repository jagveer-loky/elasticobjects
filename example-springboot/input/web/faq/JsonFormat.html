Why JSON Serialization not using Type Attribute?

There are other format extension for JSON like JSON-LD. They use an attribute "@type": "Person".

When deserializing the JSON formatted String the disadvantage is that one don't know the type until the attribute will be interpreted. There is a back reference from a field to the object.

Furthermore there is no possibility to use the typed path in the java methods.
