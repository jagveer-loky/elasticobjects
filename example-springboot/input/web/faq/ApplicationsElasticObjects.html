What are the applications of EO

From a java code perspective on can use EO just as an object manipulation tool. The size of the core library with 270 KByte is rather small.

One can

  • serialize
  • deserialize
  • compare
  • convert
  • merge

simple java objects.

With the #{ConfigLinkCall->ModelConfig, Call}. concept and the combination of calls one can create applications for almost any requirements.

There are two applications I use to check practicability:

  • ⊚ This web site elasticobjects.org by #{GithubLinkCall->NONE, B||Builder.java}.
  • ⊚ Code generation in module example-springboot with #{GithubLinkCall->NONE, SP||Start.java}.

These application takes heave use of the template calls:

  • #{ConfigLinkCall->ModelConfig, TemplateCall}.
  • #{ConfigLinkCall->ModelConfig, TemplateResourceCall}.
  • #{ConfigLinkCall->ModelConfig, TemplateResourceStoreCall}.

Core of the template call is the #{GithubLinkCall->NONE, EO|calls.templates|Parser.java}. class.
