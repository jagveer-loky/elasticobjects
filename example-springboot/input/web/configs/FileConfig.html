≡FileConfig

The file configurations allows the access to predefined files on the server.

All Files named #{GithubLinkCall->NONE, SPR||FileConfig.json}. will be loaded from the classpath into the configuration cache #{GithubLinkCall->NONE, EO|models|EOConfigsCache.java}..

In the spring boot example this is initialized by #{GithubLinkCall->NONE, SP|web|EOConfigCached.java}..

Depending on the ConfigCacheKey different configuration classes with different methods will be stored.

  • #{GithubLinkCall->ModelConfig, FileConfig}.
  • #{GithubLinkCall->ModelConfig, CsvConfig}.
  • #{GithubLinkCall->ModelConfig, XlsxConfig}.

The configuration methods will be used by the different file calls.

  • #{GithubLinkCall->ModelConfig, FileReadCall}.
  • #{GithubLinkCall->ModelConfig, FileWriteCall}.
  • #{GithubLinkCall->ModelConfig, CsvSimpleReadCall}.
  • #{GithubLinkCall->ModelConfig, FileReadCall}.
  • #{GithubLinkCall->ModelConfig, TemplateResourceCall}.
  • #{GithubLinkCall->ModelConfig, XlsxReadCall}.
