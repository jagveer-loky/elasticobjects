≡ModelConfig

The model configurations offer methods for accessing, setting or creating java objects via EO.

When the cache object #{GithubLinkCall->NONE, EO|models|EOConfigsCache.java}. is initialized any file in the classpath named #{GithubLinkCall->NONE, EOR||ModelConfig.json}. will be loaded.

In the spring boot example this is initialized by #{GithubLinkCall->NONE, SP|web|EOConfigCached.java}..

Each entry contains information about

  • ≡Field names
  • Object access
  • Class generation
  • Database constraints

The cache then contains #{GithubLinkCall->NONE, SP||EOConfigMapModels}. model map with entries of the following types:

  • #{ConfigLinkCall->ModelConfig, ModelConfigScalar}.
  • #{ConfigLinkCall->ModelConfig, ModelConfigList}.
  • #{ConfigLinkCall->ModelConfig, ModelConfigMap}.
  • #{ConfigLinkCall->ModelConfig, ModelConfigObject}.

With superKey and interfaces fields it contains the necessary information about inheritance structure.
