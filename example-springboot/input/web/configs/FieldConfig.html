The field configurations provide all necessary informations about the type of a field

Any #{GithubLinkCall->NONE, EOR||FieldConfig.json}. file from the classpath is loaded together with the ≡model configurations into the configuration cache #{GithubLinkCall->NONE, EO|models|EOConfigsCache.java}..

In the spring boot example this is initialized by #{GithubLinkCall->NONE, SP|web|EOConfigCached.java}..

The separation of field and ≡model configurations allow to avoid annoying, misleading and error prone definitions of fields for each model with the same name.

When the model is initialized via #{GithubLinkCall->NONE, EOTTR||Models.json}. class names, the fields will be added as [className].[fieldName].
