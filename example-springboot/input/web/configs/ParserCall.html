≡TemplateConfig

The template configurations allows the access to predefined templates on the server.

Templates are parsed by #{GithubLinkCall->NONE, EO|calls.templates|Parser.java}. for

  • $[<path>]
  • $[@<system variable>]
  • $[%<environment variable>]
  • $[(<call)<path <attributes>]
Special shortages for special calls are
  • TemplateCall: $[&<path <attributes>]
  • ValueCall: $[#<path <attributes>]

Templates allows a calling hierarchy returning a valuable content or just call things on the server. The json calls only allow a subsequent execution.

All Files named #{GithubLinkCall->NONE, SPR||FileConfig.json}. will be loaded. will be loaded from the classpath into the cache object #{GithubLinkCall->NONE, EO|models|EOConfigsCache.java}. the first time a template call will be initiated.

All pages in the examples are created by two templates:

  • #{ConfigLinkCall->FileConfig, ContentPage.html}.
  • #{ConfigLinkCall->FileConfig, ConfigsPage.html}.
This page is generated via #{ConfigLinkCall->FileConfig, ContentPage.html}.

On each page you will see a form with a text field containing a json call to create the same page via the generic post-call by eo-form #{GithubLinkCall->NONE, SP|web|WebEo.java}. and not with the get endpoints #{GithubLinkCall->NONE, SP|web|WebEoGet.java}. with url path variables.
