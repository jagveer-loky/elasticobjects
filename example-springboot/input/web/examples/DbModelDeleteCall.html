Delete Example

With #{ConfigLinkCall->ModelConfig, DbModelDeleteCall}. one has the possibility to delete entries in the database on the server side depending on the object parameters defined in #{ConfigLinkCall->FieldConfig, sourcePath}..

In the example the #{ConfigLinkCall->FieldConfig, sourcePath}. "abc" contains an #{ConfigLinkCall->ModelConfig, AnObject}. object with #{ConfigLinkCall->FieldConfig, naturalId}. "key2". #{ConfigLinkCall->ModelConfig, DbModelDeleteCall}. will remove the second entry of the preloaded #{GithubLinkCall->NONE, EODBR||DbSqlConfig.json#L41}.. The deleted entry will be returned on #{ConfigLinkCall->FieldConfig, targetPath}. "xyz".

The examples would not work since the role guest has not the permission to delete. Only #{ConfigLinkCall->ModelConfig, DbModelReadCall}. is allowed.

#{ConfigLinkCall->ModelConfig, AnObject}. shows that there is a only read property for "guest".
