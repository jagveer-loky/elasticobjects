Read Example

With #{ConfigLinkCall->ModelConfig, DbModelReadCall}. one has the possibility to execute generic database queries on the server side depending on the object parameters defined in #{ConfigLinkCall->FieldConfig, sourcePath}..

In the example the #{ConfigLinkCall->FieldConfig, sourcePath}. "abc" contains an #{ConfigLinkCall->ModelConfig, AnObject}. object with #{ConfigLinkCall->FieldConfig, id}.=1. #{ConfigLinkCall->ModelConfig, DbModelReadCall}. will query the first entry of the preloaded #{GithubLinkCall->NONE, EODBR||DbSqlConfig.json#L40}. and store the result to the #{ConfigLinkCall->FieldConfig, targetPath}. "xyz".

#{ConfigLinkCall->ModelConfig, AnObject}. shows that there is a read property for "guest".

So the examples for #{ConfigLinkCall->ModelConfig, DbModelWriteCall}. or #{ConfigLinkCall->ModelConfig, DbModelDeleteCall}. won't work actually, since you have the role guest with your request and no permission to execute.

Actually no object mapping of the result is provided. The raw query result will just returned as a map by default.
