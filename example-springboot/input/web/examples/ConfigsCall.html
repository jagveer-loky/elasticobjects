Configuration Calls

That a client knows the configurations a server provide, it's important to deliver information about the provided configurations. Therefore the config calls are created.

  • #{ConfigLinkCall->ModelConfig, ConfigTypesCall}.: Lists the keys of available configuration types
  • #{ConfigLinkCall->ModelConfig, ConfigKeysCall}.: Lists all naturalIds for a given "configType" matching "configFilter"
  • #{ConfigLinkCall->ModelConfig, ConfigCall}.: Lists complete configurations for a given "configType" matching "configFilter"

Since these calls just deliver loaded configurations from the configuration cache, no other informations like permission restriction are needed. The configuration calls are plain value calls like the TheGreetingCall with a model config without configuration by its own.

On every configuration page calls are executed to get informations about the loaded configurations.

ConfigKeysCall

#{ConfigLinkCall->ModelConfig, ConfigKeysCall}. returns the "naturalId" of matching configurations.

In the following example all ModelConfig keys will be listed matching #{ConfigLinkCall->FieldConfig, configFilter}. "Config.*Call".

#{ConfigLinkCall->FieldConfig, ConfigType}. must be set with a existing configuration class.

ConfigCall

#{ConfigLinkCall->ModelConfig, ConfigCall}. returns the complete configurations.

In the following example all ModelConfig configurations will be listed matching the #{ConfigLinkCall->FieldConfig, configFilter}. "Config.*Call".

ConfigTypesCall

With #{ConfigLinkCall->ModelConfig, ConfigTypesCall}. all available configType keys will be listed.
