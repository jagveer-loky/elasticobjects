FileCalls

One can give a remote user access to a file when defining some configurations in FileConfig.json.

Here some examples of reading file content to EO with the following two files:

  • #{ConfigLinkCall->FileConfig, public.txt}.
  • #{ConfigLinkCall->FileConfig, secret.txt}.

public.txt

In the following example a path "xyz" will be specified of type "FileReadCall".

If you click, you will see the content is set to "xyz" since your role "guest" with the generic interface is in the read permissions.

If you remove "_serializationType" you can see the hidden fields used by EO, for example the duration of this call.

All values in the curly brackets of "xyz" will be mapped to the field values of #{ConfigLinkCall->ModelConfig, TheGreetingCall}..

secret.txt

In the following example a path "xyz" will be specified of type "FileReadCall" with a "fileConfigKey" with value "secret.txt" where the permissions for "read" is "admin.

If you click, you will see the content is set to "xyz" with an error message, since you have only the role "guest" with the generic interface.

A file with no permissions

The example file public.txt there is another configuration without any permission. If you call this configuration by the generic interface, you would have no permissions.

If you click, you will see the content is set to "xyz" with an error message.
