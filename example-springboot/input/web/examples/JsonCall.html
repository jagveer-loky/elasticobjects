Json Calls

One can give a remote user access to a json file which content will automatically parsed, when defining an configurations entry in ≡FileConfig.json.

Here some examples of reading file content to EO with the following two files:

  • #{ConfigLinkCall->FileConfig, list.json}.
  • #{ConfigLinkCall->FileConfig, map.json}.

map.json

In the following example a path "x/y/z" will be specified of type "FileReadCall".

If you click, you will see the values of #{ConfigLinkCall->FileConfig, map.json}. is set to "x/y/z" since your role "guest" with the generic interface is in the read permissions.

If you remove "_serializationType" you can see the hidden fields used by EO, for example the duration of this call.

map.json as Object

In the following example a path "xyz" will be specified of type "FileReadCall" with the targetPath.

If you click, you will see the content is set as the object AnObject.
